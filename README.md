# TimeTravelAgency - Rapport de projet

## 1. Presentation du projet

TimeTravelAgency est une webapp fictive de voyage dans le temps. Le concept simule une agence premium qui propose des destinations historiques immersives, avec une interface multilingue, une navigation visuelle riche et un assistant conversationnel integre.

Le projet presente plusieurs epoques emblematiques, comme Paris 1889, Florence 1504, la Rome antique, l'epoque d'Edo, l'ere du Cretace ou encore l'arrivee sur la Lune avec Apollo 11.

## 2. Objectif du projet

L'objectif principal etait de concevoir une experience web narrative et interactive autour d'un univers fictif, tout en mettant en avant :

- une presentation haut de gamme;
- une navigation fluide entre les sections;
- une interface multilingue;
- un assistant conversationnel en front-end;
- une structure technique moderne et maintenable.

## 3. Technologies utilisees

Le projet repose sur les technologies suivantes :

- Next.js 16 avec React 19
- TypeScript
- Tailwind CSS 4
- Framer Motion pour les animations
- Lucide React pour les icones
- shadcn/ui et Base UI pour les composants UI
- next/font pour la gestion typographique
- Vercel Analytics pour le suivi de visite

## 4. Fonctionnalites implementees

Les principales fonctionnalites developpees sont :

- une page d'accueil immersive avec hero anime;
- une navigation principale claire et responsive;
- une galerie de destinations temporelles;
- une page detaillee pour chaque destination;
- un filtre et une recherche sur la page destinations;
- un chatbot conversationnel nomme Chronos;
- une section AI Preview qui simule des echanges de recommandation;
- un quiz permettant d'orienter l'utilisateur vers une destination adaptee;
- une page de contact pour initier une demande de reservation;
- un footer complet avec newsletter et liens utiles.

## 5. Utilisation de l'IA

Pour respecter la transparence du projet, voici les usages lies a l'IA :

- v0 de Vercel a ete utilise pour accelerer le scaffolding initial et la production de l'interface;
- le chatbot Chronos est implemente localement avec une logique basee sur des mots-cles;
- Utilisation de GitHub Copilot pour la documentation.

## 6. Instructions d'installation

Prerequis : Node.js et un gestionnaire de paquets compatible avec le lockfile, idealement pnpm.

```bash
pnpm install
pnpm dev
```

Le projet sera ensuite accessible sur [http://localhost:3000](http://localhost:3000).

Commandes complementaires :

```bash
pnpm build
pnpm start
pnpm lint
```
